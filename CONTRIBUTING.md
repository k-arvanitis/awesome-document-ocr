# Contributing to Awesome Document OCR

Suggestions and improvements are welcome.

## How to Add an Entry

1. Fork this repository.
2. Add your entry to the relevant section in `README.md`.
3. Use the existing table format (Link, Type, License, Output, Description, Strengths).
4. Open a pull request with a short explanation of why the tool belongs here.

## Entry Criteria

- The tool must be actively maintained (last commit within 12 months, or widely used).
- Description must be objective — no marketing language.
- Link must point to the official GitHub repo or project page.
- License type must be accurate (Open Source / Open Weights / Closed API).

## What Does Not Belong

- Unmaintained or archived projects.
- Tools already covered by a listed entry.
- General-purpose ML frameworks with no specific document/OCR focus.

## Format

Follow this exact structure for each entry:

```markdown
### Tool Name

| Field | Detail |
|---|---|
| **Link** | [domain/path](https://url) |
| **Type** | e.g. VLM OCR, Document Parser, Pipeline |
| **License** | Open Source / Open Weights / Closed API |
| **Output** | e.g. Markdown, JSON, structured text |

**Description:** One or two sentences. Objective, copy-paste ready.

**Strengths:** Comma-separated list of key strengths.
```
